1. Layout

When creating your layouts use white space to create a visual hierarchy on the page, aim to design for small screens first, using a single column layout. Consider how you can optimise your layout for different screen sizes, but don’t make assumptions about what devices people are using (things change.)

  • See an example of GOV.UK grid layout

Spacing

  • Spacing between elements is usually 5px, 10px, 15px or multiples of 15px.
  • Gutters are 15px for smaller screens and 30px for larger screens.

Grid unit proportions

  • Base column ratios on halves, thirds or quarters of the page width.
  • Set breakpoints for small, medium and large screens using media queries.
  • The media queries for these breakpoints can be found in the GOV.UK frontend toolkit _conditionals.scss file.
  • Introduce columns as the content requires it.

3. Colour

Colour and accessibility

When using colour on text and functional elements (like buttons) make sure difference between the foreground and background has a high enough in contrast to be seen by those with visual impairments. We aim to meet the AA colour contrast standard ratio of 4.5:1 prescribed by W3C, to do this we recommend testing using colour contrast tools.

Text, background and button colours

  • Use Sass variables below in case colour values need to be updated, the variables are defined in colours.scss of the GOV.UK front end toolkit.
  • Text should be set in black $text-colour with secondary text in grey $text-secondary.
  • Links should be set in blue $link-colour and buttons or dialog boxes should have text in white.

4. Images and icons

Avoid unnecessary decoration, only use images and icons if there is a there is a real user need.

  • See an example of image usage on GOV.UK

Icons

  • If icons are needed ensure they are clear, simple and help to convey information more clearly than text alone could. Don’t hide functionality under icons.
  • Ensure icons meet contrast ratio requirements.

Image aspect ratio

If images are needed they should be landscape, 3:2 aspect ratio.

6. Forms

Keep forms as simple as possible, only ask what is needed to run your service. All elements use the yellow focus style as a highlight, as either a fill or 3px outline. The focus colour is Sass variable $yellow, Hex #FFBF47.

See an example of GOV.UK form layout

Dropdowns

Avoid using dropdowns, instead we recommend; radio buttons, using predictive fields (good for long lists like countries) and checkboxes.

Form inputs and labels

Labels should be aligned above their controls.

Spacing between form elements

Ensure there is sufficient spacing between form elements.
Use .form-group to group label and input pairs and create 45px bottom spacing.

As shown on your birth certificate or passport

Hint text and example text

  • Use hint text for supporting contextual help.
  • Don’t use placeholder text in fields.
  • Hint text should sit above a form field.

It'll be on your last payslip. For example JH 219 00A.

Radio buttons

  • Radio buttons let users choose a single option from a list.
  • For two options, these can sit side-by-side.
  • For more than two options, radio buttons or checkboxes should be stacked.
  • Radio buttons with large hit areas are easier to select with a mouse and also for touch devices.

Checkboxes

  • Checkboxes let users select either on/off toggles or multiple selections.
  • Checkboxes with large hit areas are easier to select with a mouse and also for touch devices.

7. Buttons

Use buttons to move though a transaction. Don't use them to represent choices, instead use radio buttons, checkboxes or links.

Button colours

  • Buttons use the use the Sass variable, $button-colour, Hex #00823B.
  • Buttons can be created, by passing the above colour variable to the button mixin defined in the GOV.UK front end toolkit.
  • All buttons use the Sass variable $yellow, Hex #FFBF47 as a focus colour, often this is used as a fill or an outline of 3px.

8. Errors and validation

Have a summary box with anchor links, which jump down to the relevant section of the form where the error is highlighted.

See an example of form validation on GOV.UK

Form validation errors

  • Summarise validation errors in a box at the top of your page.
  • Use a red background and border on the box to visually connect the errors and messages in the form.
  • Apply the same colour to the area of the form where an error has occurred.
  • Use jump links in the summary box, to reach the errors in the form.
  • Error copy should be specific to the question and validation should identify all errors.
  • Errors should not cause pre-filled fields to clear.

  • Enter your email address

Confirmation pages

All new and redesigned services should provide transaction end pages.

Find out more about confirmation pages

Transaction end pages:

  • tell the user that their transaction has finished OR
  • tell the user what they have to do next (if they're not finished)
  • prompt the user to give feedback on the service – to measure user satisfaction
  • count completed transactions – to measure completion rate

Transaction end pages are hosted by the service.

Discuss transaction end pages on the Design Patterns Hackpad.
